# Bitácora de cambios

Este documento registra los avances principales del proyecto Noti Inclusivos.

La bitácora permite dejar evidencia del desarrollo progresivo del boletín, diferenciar versiones iniciales de futuras mejoras y explicar el sentido educativo y comunicacional de los cambios realizados.

## Versión inicial

Se crea el repositorio Noti Inclusivos como boletín digital educativo personal para organizar y difundir publicaciones, orientaciones, capacitaciones y recursos accesibles dirigidos a docentes, familias y comunidad educativa.

Se amplía el archivo `README.md` con finalidad del proyecto, público objetivo, alcance educativo y comunicacional, contenidos previstos, estructura sugerida, autoría, licencia y estado de avance.

Se incorpora el archivo `LICENSE` para establecer condiciones de uso del código y aclarar que los materiales externos o de terceros mantienen sus propias condiciones de uso.

Se agregan documentos en la carpeta `docs` para organizar autoría, alcance comunicacional, fuentes, créditos, uso permitido, criterios de publicación, estructura del proyecto y bitácora de cambios.

## Pendientes de desarrollo

- Definir secciones del boletín.
- Organizar publicaciones por categorías.
- Incorporar publicaciones y recursos con fuente clara.
- Revisar accesibilidad visual y navegación.
- Agregar descripciones a imágenes o materiales visuales.
- Registrar fuentes externas y condiciones de uso.
- Diferenciar publicaciones educativas, orientativas y de sensibilización.
- Actualizar la bitácora cuando existan cambios importantes.

## Registro de cambios futuros

Cada mejora significativa debe anotarse con fecha, descripción breve y motivo del cambio.

```text
Fecha:
Cambio realizado:
Motivo:
Responsable:
Observaciones:
```

Este registro no reemplaza los commits de GitHub, pero ayuda a documentar el sentido comunicacional, educativo y organizativo de cada modificación importante.
