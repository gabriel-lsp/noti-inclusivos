# Bitácora de cambios

Este documento registra los avances principales del proyecto Noti Inclusivos.

La bitácora permite dejar evidencia del desarrollo progresivo del boletín, facilitar la revisión institucional y diferenciar las versiones iniciales de futuras mejoras.

## Versión inicial

Se crea el repositorio Noti Inclusivos como boletín digital educativo para organizar y difundir comunicados, orientaciones, capacitaciones y recursos accesibles dirigidos a docentes, familias y comunidad educativa.

Se amplía el archivo `README.md` con finalidad del proyecto, público objetivo, alcance educativo y comunicacional, contenidos previstos, estructura sugerida, autoría, licencia y estado de avance.

Se incorpora el archivo `LICENSE` para establecer condiciones de uso del código y aclarar que los materiales institucionales o de terceros mantienen sus propias condiciones de uso.

Se agregan documentos de respaldo en la carpeta `docs` para sustentar el alcance comunicacional, fuentes, créditos, uso permitido, respaldo institucional, criterios de publicación y estructura del proyecto.

## Pendientes de desarrollo

- Definir secciones del boletín.
- Organizar publicaciones por categorías.
- Incorporar comunicados y recursos con fuente clara.
- Revisar accesibilidad visual y navegación.
- Agregar descripciones a imágenes o materiales visuales.
- Registrar fuentes externas y autorizaciones de uso.
- Diferenciar publicaciones institucionales, orientativas y de sensibilización.
- Actualizar la bitácora cuando existan cambios importantes.

## Criterios para registrar cambios futuros

Cada mejora significativa debe anotarse con fecha, descripción breve y motivo del cambio.

Formato sugerido:

```text
Fecha: día/mes/año
Cambio realizado:
Motivo:
Responsable:
Observaciones:
```

Este registro no reemplaza los commits de GitHub, pero ayuda a documentar el sentido comunicacional, educativo e institucional de cada modificación importante.
